Offsetting and Ordering Columns

Two techniques for controlling column position beyond simple left-to-right flow: offsets add blank space to the left of a column; push/pull reorders columns visually without changing the HTML source order.

Offsetting — col-md-offset-N adds N empty columns of space to the left

Row 1: col-md-5 + col-md-6 = 11 columns. The gap on the right is just unused space (1/12 of the row).

col-md-5

Five columns wide. No offset.

col-md-6 (no offset)

Six columns wide, sits immediately after col 1. 5+6=11, leaving 1 empty on the right.

Row 2: same columns, but col-md-offset-1 pushes the second column one unit right. Now 5+1+6=12 — fills the full row with a deliberate gap.

col-md-5

Five columns wide. No offset.

col-md-6 col-md-offset-1

Offset by 1 column. The dashed yellow gap to the left is the offset space.

Ordering — col-md-push-N and col-md-pull-N swap column order visually

In the HTML, the wide column (8) comes first and the narrow column (4) comes second. Push/pull lets you flip them on screen without changing the source order — useful for SEO or accessibility where content order matters.

Before push/pull — HTML order matches visual order:

HTML order: 1st. col-md-8 — wide column on the left.
HTML order: 2nd. col-md-4 — narrow column on the right.

After push/pull — same HTML order, but columns swap visually:

HTML order: 1st, but visually RIGHT. col-md-push-4 shifts this 4 units to the right.
HTML order: 2nd, but visually LEFT. col-md-pull-8 shifts this 8 units to the left.
